## 1. 專案概述

### 1.1 背景與動機

現代人每天面對大量瑣碎的日常任務，卻缺乏有效的工具將「完成任務」這件事轉化為可見的成長數據。傳統待辦清單只能打勾、無法給予成就感，導致使用者容易中途放棄。

本系統透過「成就積分」機制，讓每一個完成的任務都能累積成具體數字，幫助使用者建立完成事項的正向回饋循環，提升執行力與成就感。

### 1.2 目標用戶

- **主要族群**：一般大眾，尤其是需要管理日常瑣事的學生、上班族、自由工作者
- **技術程度**：無需特殊技術背景，操作直覺簡單
- **使用情境**：每天開啟系統確認今日任務、完成後標記、查看累積積分

### 1.3 核心價值主張

> 「把每一件瑣事，變成你前進的數據。」

- **輕量化**：無需複雜設定，開啟即用
- **成就感驅動**：完成任務即獲得積分，積累可視化成長
- **直覺操作**：清單式介面，新增、編輯、刪除一目了然

### 2.2 使用者故事

#### F-01：新增任務

> 作為 **一般用戶**，我希望 **能夠快速輸入一筆新任務**，以便 **記錄我今天要完成的事項**。

**驗收條件**：
- 使用者可在輸入框填寫任務標題後按下「新增」按鈕
- 系統將任務儲存至資料庫並立即顯示於清單中
- 新增成功後輸入框清空，等待下一筆輸入

---

#### F-02：編輯任務

> 作為 **一般用戶**，我希望 **能夠修改已建立的任務內容**，以便 **更正錯字或調整任務描述**。

**驗收條件**：
- 每筆任務旁有「編輯」按鈕
- 點擊後可修改任務標題（與選填的描述、積分值）
- 儲存後清單即時更新，不需重新整理頁面

---

#### F-03：刪除任務

> 作為 **一般用戶**，我希望 **能夠刪除不再需要的任務**，以便 **保持清單的整潔**。

**驗收條件**：
- 每筆任務旁有「刪除」按鈕
- 點擊後顯示確認提示（避免誤刪）
- 確認後任務從資料庫中移除並從清單消失
- 若任務已完成並獲得積分，刪除後積分不扣回（歷史保留）

---

#### F-04：標記任務完成狀態

> 作為 **一般用戶**，我希望 **能夠標記任務為「已完成」**，以便 **追蹤我今天做了哪些事**。

**驗收條件**：
- 每筆任務有勾選框（checkbox）
- 勾選後任務顯示劃線樣式，視覺上與未完成任務區隔
- 完成狀態儲存至資料庫，重新整理後仍保留
- 可取消勾選，狀態恢復為未完成（積分對應扣回）

---

#### F-05：成就積分系統

> 作為 **一般用戶**，我希望 **每次完成任務都能獲得積分**，以便 **感受到具體的進步與成就感**。

**驗收條件**：
- 每筆任務預設積分為 10 分，使用者可於新增/編輯時自訂（1–100 分）
- 完成任務時積分累加至用戶總積分
- 首頁顯示目前總積分
- 取消完成時扣回對應積分

## 3. 非功能需求

### 3.1 技術限制

| 項目 | 規格 |
|------|------|
| 後端框架 | Python Flask |
| 模板引擎 | Jinja2 |
| 資料庫 | SQLite（本地儲存） |
| 前端 | HTML + CSS + 少量 JavaScript |
| 部署環境 | 本地執行（localhost），無需雲端部署 |

### 3.2 效能考量

- 頁面載入時間應在 1 秒內完成（本地環境）
- 任務清單在 500 筆以內時，操作應流暢無卡頓
- 資料庫查詢應使用索引優化常用查詢欄位（如：完成狀態、建立時間）

### 3.3 安全考量

- 使用 SQLAlchemy ORM 或參數化查詢，防止 SQL Injection
- 使用者輸入內容需做基本長度限制（任務標題 ≤ 100 字元）
- 刪除操作需要二次確認

### 3.4 可用性需求

- 介面需支援桌面瀏覽器（Chrome、Firefox、Edge）
- RWD 響應式設計，支援手機瀏覽（選配）
