# PRD - 食譜收藏系統

## 1. 專案概述
- **背景與動機**：在日常生活中，經常會看到不錯的食譜想要收藏起來，或者臨時想用手邊現有的食材做成料理，卻不知道可以搭配什麼食譜。因此需要一個專屬的「食譜收藏系統」來有效紀錄、整理與尋找食譜。
- **目標用戶**：我自己（個人使用的食譜管理與應用工具）。
- **核心價值主張**：提供一個輕量、簡單易用的系統，能快速儲存食譜、精準搜尋，並且能根據手邊現有的食材來推薦相關食譜，解決日常生活中不知道要煮什麼的困擾。

## 2. 功能需求
- **新增食譜**
   - 作為我自己，我希望可以記錄並新增食譜（包含名稱、所需食材清單、製作步驟等），以便日後查看與實作並累積專屬食譜庫。
- **搜尋食譜**
   - 作為我自己，我希望可以透過關鍵字搜尋食譜名稱或內容，以便快速找到我現在想做的特定料理。
- **根據食材推薦食譜**
   - 作為我自己，我希望可以輸入目前冰箱擁有的食材，由系統推薦對應的食譜，以便有效利用剩餘食材並減少浪費。
- **編輯與刪除食譜**
   - 作為我自己，我希望可以隨時修改食譜內容或刪除不再需要的食譜，以便保持食譜資訊的正確性與符合當下的口味需求。
- **食譜分類與標籤**
   - 作為我自己，我希望可以幫食譜加上不同的分類或標籤（例如：中式、義式、甜點、快速料理等），以便能透過分類快速瀏覽相關類別的食譜。

## 3. 非功能需求
- **技術限制**：後端採用 Flask 框架，前端使用 Jinja2 模板引擎進行渲染，資料庫使用輕量的 SQLite。
- **效能與安全考量**：
   - 介面操作反應快速流暢，提供良好的單人使用體驗。
   - 實作防範基本 SQL Injection 以及 XSS 攻擊的機制，確保輸入與輸出的資料安全。
   - 由於設定為個人單人使用系統，暫時不需要建置複雜的使用者驗證或高併發負載處理。

## 4. MVP 範圍
- **Must Have（必要功能）**：
   - 新增食譜（包含食材與步驟）。
   - 瀏覽食譜列表與詳細內容。
   - 透過關鍵字搜尋食譜。
   - 編輯與刪除食譜。
- **Should Have（重要功能）**：
   - 根據現有食材推薦食譜功能。
   - 食譜的基礎分類功能。
- **Nice to Have（次要功能）**：
   - 多重標籤篩選。
   - 匯入/匯出食譜功能。
   - 食譜的圖片上傳功能。
